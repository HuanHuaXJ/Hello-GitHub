# Git 简介

## 什么是 Git？

Git 是一个**分布式版本控制系统**。它的核心功能是**管理和跟踪文件（特别是代码）的变更历史**。

您可以把它想象成一个极其强大的"文件时光机"和"协作工具"的结合体。

## 核心概念与工作原理

Git 并不简单地存储每个版本的文件副本，而是通过一系列快照来管理。理解以下几个核心概念至关重要：

### 1. 仓库 (Repository)
- 一个被 Git 管理的项目文件夹，里面包含了项目的所有文件以及 Git 的版本记录
- 仓库可以是本地的（在你自己电脑上），也可以是远程的（在服务器上，如 GitHub）

### 2. 提交 (Commit)
- 提交是 Git 的核心操作，代表一次**版本快照**
- 每次提交都会记录：**谁**、在**什么时间**、**为什么**修改了哪些内容
- 每次提交都有一个唯一的 ID（哈希值），让你可以精准地定位到任何一个历史版本

### 3. 工作区、暂存区和版本库
- **工作区**：你在电脑上直接看到和编辑的文件
- **暂存区**：一个中间区域。你可以将工作区的修改"添加"到这里，准备进行下一次提交
- **版本库**：存放所有提交历史的地方

**基本工作流程**：`工作区修改` → `git add` → `暂存区` → `git commit` → `版本库`

### 4. 分支 (Branch)
- 分支是 Git 的"杀手级"功能。它允许你从主开发线上分叉出去，在不影响主线的同时进行工作
- 你可以为新功能、问题修复或实验性的想法创建独立的分支
- 完成后，再将分支合并回主线

## 为什么 Git 如此重要？

1. **版本回溯**：任何时候都可以回退到任何一个历史版本
2. **协作开发**：多人可以在同一个项目上工作，通过分支和合并来整合代码
3. **责任明晰**：每一次修改都有记录，谁在什么时候改了哪行代码一清二楚
4. **分支管理**：轻松地尝试新想法而不破坏稳定代码

## Git 与 GitHub 的关系

这是一个常见的误解，需要厘清：

- **Git**：是一个**工具**，一个在你的本地电脑上运行的**版本控制软件**
- **GitHub**：是一个基于 Git 的**网站和云端服务平台**

**简单比喻**：
- **Git** 像是你电脑上的单机版 Word 软件，负责文档的编辑和本地保存
- **GitHub** 像是 Google Docs，它让你可以把用 Word（Git）写的文档上传到云端，并方便地与其他人共享和协作

---

**总结**：Git 是一个强大、高效、分布式的版本控制系统，它通过记录文件的每一次变更、提供强大的分支模型，彻底改变了软件开发和协作的方式。
